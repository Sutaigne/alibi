1 · What this is

A read-only Windows scanner that produces credible proof

Someone asked you to demonstrate that your gaming setup isn't running cheats. This kit lets you do that. You run a scan on your machine. It writes four files to your Desktop. You send those files. The reviewer reads them and decides for themselves.

Alibi is read-only. It does not connect to the internet. It does not install anything. It does not change any system setting. It runs once, writes the report files, and exits. Total runtime: about 2-3 minutes.

2 · What's in the files

Two things you touch. Everything else is the engine.

When you open the folder, you see three things: a text file telling you what to do, a launcher to double-click, and a "scanner" subfolder containing everything that runs behind the scenes. You only ever interact with the first two.

You interact with these

Top level

START HERE.txt
A short readable intro. Open it first if you want to know what you're looking at before clicking anything.
Run scan.bat
The launcher. Right-click → Run as administrator. Both scans run automatically. The window shows progress and stays open at the end so you can read the summary.
scanner\
The engine subfolder. You don't need to open anything in here — the launcher handles it. Reference docs live here for the reviewer if they want them.
Inside the scanner\ subfolder

The engine + reference docs

forensic-common.ps1
The shared scan engine. ~900 lines of plain PowerShell. Contains every keyword the scan looks for and every utility function.
forensic-scan.ps1
Runs the PC-mode scan. Looks for PC-side cheats, DMA artifacts, HWID spoofers.
console-rig-audit.ps1
Runs the console-rig-mode scan. Looks for console-MITM cheat stacks (vision aimbots, input adapters).
generate-visual-companion(-console).ps1
Renders the .txt reports into HTML visuals so they're easier to read.
alibi-safety-card.html
Safety attestation. Open this if you're the reviewer and want a one-page verification of what the scan does and doesn't do.
README.txt + other explainers
Reference documentation, deeper explainers for specific topics (DMA, console lockdown, etc.).

Every .ps1 file in scanner\ is plain text. Right-click any of them and Open with → Notepad to read every line the scan runs. There is no compiled binary, no installer, no obfuscation.

3 · How to use it

Four steps, no decisions.

1

Download and unzip the folder

Put it anywhere — Desktop, Downloads, a USB stick. The folder structure must stay intact (the Run scan.bat looks for the scanner\ subfolder next to it).

2

Right-click Run scan.bat → "Run as administrator"

Windows will show a UAC prompt asking for admin permission. Click YES. The scan needs admin to read several Windows forensic sources (Prefetch, BAM, USB history, etc.). Without admin most of these come back blank, and the report is too incomplete to be useful.

3

Wait about 2-3 minutes

Both scans run automatically — first the PC-mode scan, then the console-rig-mode scan. The black window shows progress messages. Do not close it.

4

Send the four output files

When the scan finishes the window stays open with a "FINAL SCAN SUMMARY" block showing both verdicts. Four files appear on your Desktop:

AlibiReport_<timestamp>.txt
AlibiReport_<timestamp>_visual.html
AlibiRigReport_<timestamp>.txt
AlibiRigReport_<timestamp>_visual.html

Send the .txt files (or the .html visuals — both contain the same data) to whoever asked.

There is no menu, no configuration, no decisions. The launcher runs both modes automatically because most setups benefit from both. If you're a pure PC gamer the console report will be CLEAN (expected); if you're a pure console gamer with a PC in your rig the PC report will be CLEAN (also expected). The relevant report is whichever has findings.

4 · Reading the report

What the .txt and .html files actually contain

Each report has the same structure. The first thing you see is a QUICK READ block at the top with the verdict and named items. Below that are three detailed sections: cheat-trace findings, scored running processes, and scored services. The _visual.html companion shows the same data as charts and tables.

What the top of every .txt looks like:

================================================================ QUICK READ - START HERE ================================================================ VERDICT: CLEAN No HIGH or MEDIUM matches against the cheat / input-device / dual-use keyword database. Scope of this scan: Cheat-trace findings checked : 1247 total artifacts Running processes scored : 232 Services scored : 491 This is necessary but not sufficient evidence. See limitations section at the bottom of this report for what this scan cannot detect. ================================================================

What an individual finding looks like (if any are flagged):

[HIGH/cheat] [Prefetch] [pcileech] PCILEECH.EXE-A1B2C3D4.pf Source: C:\Windows\Prefetch\PCILEECH.EXE-A1B2C3D4.pf FirstSeen: 2025-12-04T18:22:10 LastModified: 2026-04-11T03:15:42 Pattern: pcileech

Each finding says: severity (HIGH/MEDIUM), category (cheat/input/dual-use), what scan source matched it (Prefetch, BAM, USB, etc.), what keyword pattern matched, where the evidence was found on disk or in the registry, and any relevant timestamps.

The verdict tier glossary

CHEATS DETECTED PC-mode found a HIGH-confidence keyword match for a cheat brand, HWID spoofer, or DMA artifact. The report names exactly what and where.
INPUT DEVICES DETECTED PC-mode found HIGH-confidence matches for XIM / Cronus / ReaSnow / KMBox / reWASD configurator software. Whether these are cheats depends on the game's rules.
MITM CHEAT STACK DETECTED Console-rig-mode found the canonical console-cheat fingerprint: vision-aimbot software, input-adapter configurators, or DMA artifacts on the rig PC.
CAPTURE STACK PRESENT Console-rig-mode found capture-card software (Elgato, OBS) or HID-emulation drivers (ViGEmBus, DS4Windows) but no aimbot or adapter software. Usually means: legit streamer setup, not a cheater.
UNSURE No HIGH matches but at least one MEDIUM. Typically dual-use tools (Cheat Engine, x64dbg, BleachBit) or binaries in user-writable locations the allowlist doesn't recognize. Needs human review.
CLEAN No HIGH and no MEDIUM matches. The report still includes the full process and service tables for transparency. A clean scan is necessary but not sufficient (see Section 6).

The _visual.html companion shows the same data as a single-page browser report — with a colored verdict header, stacked-bar charts of HIGH / MEDIUM / LOW / CLEAN counts across processes and services, a flagged-items table, and an activity timeline if dated events are present. Same information, easier to scan visually.

5 · Why it's safe

What the scan does, doesn't do, and how you can verify any of it

The scan is not malware. It's not "a script that does stuff to your computer." It's a read-only forensic inspection that produces text output. Here are the six guarantees, each one independently verifiable.

✓

Read-only

The scan never modifies a file, registry key, service, driver, or system setting. It only reads.

✓

One opt-in network call, nothing else

The only outbound call is the LOLDrivers cross-reference (loldrivers.io) for BYOVD detection — you press Y to allow it, anything else to skip. No telemetry, no phone-home, no cloud upload. Nothing about your PC is sent.

✓

Plain text source

Every .ps1 file is plain PowerShell. Open any of them in Notepad and read every line. No compiled binary, no installer.

✓

Fair to old artifacts

Findings older than 180 days are logged in the report but don't bump the verdict. Someone who cheated in another game years ago and is now scanning a clean current rig shouldn't get the same result as a present-day cheater. Historical findings appear in their own section so the reviewer can still see them and make their own call.

✓

No installation

Doesn't install anything, doesn't add startup entries, doesn't create services, doesn't drop files in System32 or Program Files.

✓

Single output location

The only files written are the four reports on your Desktop. Two .txt and two _visual.html. Nothing else, anywhere.

✓

No background process

Runs once for ~2-3 minutes, writes the reports, exits. No scheduled task, no service, no startup entry. The window closes when you close it.

Verify it yourself in 60 seconds

METHOD 1

Open the source

Right-click scanner\forensic-scan.ps1 → Open with → Notepad. Search scanner\*.ps1 for Invoke-Web and you'll find exactly one result — the opt-in LOLDrivers fetch on forensic-common.ps1 line 736, prompted with Y/N at runtime. Search for Remove-Item or Set-ItemProperty and you'll find none.

METHOD 2

Watch network traffic

Open Task Manager → Performance → Ethernet (or Wi-Fi). Run the scan. The Send graph stays at zero kbps for the entire scan.

METHOD 3

Check the output

The four reports are the only things the scan creates. Open the .txt in Notepad. Every line is human-readable.

Final commitment: if something the scan does isn't visible in the source, the scan doesn't do it. There is no hidden behavior. The full attestation lives at scanner\alibi-safety-card.html.

6 · Honest limits

What it can and can't catch

No forensic tool is comprehensive. The scan catches a lot of things. It also genuinely cannot detect some others, by design. Both sides matter for the reviewer to interpret the result correctly.

What it WILL catch

Across 22 Windows forensic sources

  • Cheat brands by name (EngineOwning, PhantomOverlay, rut.gg, etc.)
  • HWID spoofers (Sync, TraceX, SlothyTech, HidHide)
  • DMA-cheat development artifacts (_top.bin, pcileech FPGA outputs)
  • Input adapters (XIM, Cronus, ReaSnow, KMBox, reWASD)
  • Vision aimbots (AimMMO, AimFlux, NoRecoilZ, AimSync)
  • LUA / AutoHotkey anti-recoil scripts (Logitech G HUB and Razer Synapse macros)
  • Cheat-named Lua scripts in user folders (aimbot.lua, esp.lua, bhop.lua, etc.)
  • DLLs injected into running game processes
  • Historical DLL-injection events (Sysmon EID 7, Security EID 4688, Prefetch — Xenos, Extreme Injector, Syringe, etc.)
  • DDoS / network-attack tools (LOIC, HOIC, slowloris, GoldenEye, torshammer, hulk, rudy, ufonet)
  • BYOVD — vulnerable and malicious drivers (opt-in LOLDrivers cross-reference against loldrivers.io; how DMA and kernel-level cheats most often get ring-0 access)
  • PC-side AI-vision aimbots (Aimmy, sunone_aimbot, RootKit AI-Aimbot, Zelesis NEO, Unibot, Embedded-AI Pi) — the up-and-coming category that runs locally with YOLO/ONNX models + virtual HID
  • Per-game cheat brands — CS2 (Neverlose, Memesense, Fatality, Primordial, Skeet, Onetap, Aimware, Axion), Apex (Kernaim, CosmoCheats), Tarkov (Phantom EFT, CheatVault), Rust (Cobra, Atomic, Cheater.Ninja), R6 (HyperForce), Marvel Rivals (Marvel Maxim, EloCarry)
  • Branded DMA hardware vendors beyond raw pcileech — Atomic, Captain, Leet, Lurker, Suspect, Phoenix Labs, Squirrel, Enigma X-1, MVP, HackDMA, ZDMA, Captain Fuser
  • Obscured filenames (hex, numeric, ultra-short)
  • Known-bad SHA256 hashes (curated database)
  • Anti-cheat-killer batch scripts (taskkill /f /im EasyAntiCheat)
  • Defender-disabler patterns inside .bat / .ps1 files
  • Driver-signing bypass flags (bcdedit /set testsigning)
  • USB device history with first-install / last-use timestamps
  • Anti-forensic patterns (event-log clearing, shadow-copy deletion)
What it WON'T catch

Real architectural blind spots

  • The console itself (PS5 / Xbox cannot be scanned — consoles are locked-down by design)
  • DMA cheats at runtime — the cheat hardware lives on a separate physical device and never touches your PC
  • Input devices configured on a separate machine and used purely as pass-through
  • Modified controllers with internal macro chips (visual inspection only)
  • Cheats brand-new enough to not be in the keyword database yet
  • Machines professionally cleaned with commercial anti-forensic tools
  • Cheats running under a different user account or service identity that admin can't enumerate
  • Session-only cheats that loaded into memory and exited without leaving disk artifacts

A clean scan is necessary, not sufficient. It raises the cost of cheating undetected substantially — most cheats leave artifacts, and the scan finds them. But it does not lower that cost to zero. The reviewer should weigh this result against everything else they know about the player.
