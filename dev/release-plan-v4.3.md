## Ship steps (in order)

1. **Wire the remote and push.** The local repo was rebuilt from the zip, so
   history diverges from the remote's 46 commits. Decision needed — either:
   - **(a) Fetch + rebase/merge** local commits onto the remote history
     (preserves the 46-commit history; the local root commit duplicates the
     v4.2.1 tree, so a rebase of the 6 post-baseline commits onto
     `origin/main` is clean *if* the baseline matches v4.2.1 exactly), or
   - **(b) Push local as-is to a branch**, PR it, and keep remote history.
   Verify with `git remote add origin … ; git fetch ; git diff origin/main 832da41`
   — if the diff is empty, (a) is safe and simple.
2. **Push `main` + tag `v4.3.0`.** CI (parse + P0 tests) must go green.
3. **Create the v4.3.0 release** from the tag. Attach **`dist\alibi.zip`**
   (SHA256 `2c825335a1c7…f062e822` at time of writing — rebuild + re-hash if
   anything changes) **and `HASHES.txt`**, matching the existing asset
   pattern so `releases/latest/download/alibi.zip` keeps working.
   Draft notes below.
4. **Update the About sidebar** (Settings → edit repo details):
   - Description (suggested): *"Someone thinks you're cheating? Prove them
     wrong. One read-only scan, four report files on your Desktop, nothing
     installed or changed. PowerShell, open source, covers CoD / CS2 / Apex /
     Tarkov / Rust / R6 / Marvel Rivals plus DMA, BYOVD drivers, and
     AI-vision aimbots."*
   - Remove the `python` topic; keep the rest.
   - Website field stays `sutaigne.github.io/alibi/`.
5. **Re-shoot the README screenshots** (`docs/screenshots/*.png`) from real
   v4.3.0 HTML output, and **regenerate the Pages examples** from the
   PowerShell renderer (they're currently Python-rendered). Until then the
   README's softened wording ("examples may lag the newest release") covers
   the gap honestly.
6. **After the release goes live:** download `alibi.zip` and the GitHub
   source zip fresh from github.com on a machine with Defender active — the
   June `Sprisky.U!cl` flag was a *cloud* verdict on the download, which
   local scans can't fully reproduce. If anything flags, submit via
   https://www.microsoft.com/en-us/wdsi/filesubmission (SECURITY.md
   documents this path).

## Draft release notes — v4.3.0

> ## v4.3.0 — trust-first credibility release
>
> This release exists because of two real field reports: a scan that returned
> **CHEATS DETECTED against alibi's own source code** (the kit sitting in
> `Downloads` matched its own keyword database), and a correct verdict buried
> under ~30 false MEDIUMs. Both are fixed.
>
> ### Fixed — false positives (the 5 P0s)
> - **Self-immunity:** alibi never scans its own files. A copy of this repo
>   under Desktop/Downloads produces zero findings; a real cheat folder still
>   fires (regression-tested).
> - **Authenticode trust:** signed binaries in user-writable paths (Opera GX
>   and friends) no longer flagged for their location alone.
> - **LOLDrivers matching is hash-first:** legitimate Windows drivers that
>   share a filename with a vulnerable one (`afd.sys`, `KslD.sys`) no longer
>   flag; SHA256 decides.
> - **Injector keyword matching is word-bounded** and skips Windows Error
>   Reporting events — no more phantom "DLL injection" findings from app
>   crashes.
> - **Unsigned-driver findings dedupe** — one finding per driver, not ~11.
>
> ### Fixed — kit integrity & docs
> - **HASHES.txt was wrong for 12 of 26 files in every previous download**
>   (line-ending mismatch at generation time) — `sha256sum -c` now passes.
> - Report banners stamped v4.3.0 (were hardcoded v4.0).
> - SECURITY.md refreshed: real observed AV detections named
>   (`Wacatac.B!ml`, `Sprisky.U!cl`) with the local-scan-clean tell;
>   supported-versions table current; launcher elevation described
>   accurately.
> - "Exactly one Invoke-Web" transparency claim no longer cites line numbers
>   that rot (the claim itself remains true and verified).
> - One-page guide: step 2 correctly says right-click → Run as administrator.
>
> ### Improved
> - The two back-to-back scans no longer look like one scan restarting —
>   scanners keep the launcher's `[ 1 of 2 ] / [ 2 of 2 ]` progress on
>   screen, and both banners are mode-labeled.
> - README and START HERE rewritten in plain language for the person actually
>   being accused; reviewer docs keep the precise tone.
> - CI now runs the P0 regression suite (14 tests) on top of the parse check.
>
> ### Verification
> Live elevated run on a real Windows 11 box with this repo deliberately in
> `Downloads`: 0 cheat HIGHs, 0 input HIGHs, 1 legitimate MEDIUM. Defender
> scans clean on the tree, the built `alibi.zip`, and a source-style zip —
> including with Mark-of-the-Web applied. 14/14 regression tests.
