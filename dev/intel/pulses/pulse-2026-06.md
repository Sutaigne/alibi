## Non-storefront test — GitHub release assets (2026-06-24)

To check whether the website/asset methods *ever* surface real software filenames (the
3 vendor storefronts gated theirs), tested the opposite class via `gh api releases`:
- **Babyhamsta/Aimmy** → `AimmyV2.5.0.zip` (102k+ downloads) — real artifact name.
- **TheRyoSaeba/BLOPS7** (BO7 DMA) → `Release.rar`.
- IntelSDM/Cod4DMA, lauralex/xDefiantDMA → source-only, no releases.

**Conclusion:** the method DOES point to meaningful filenames on the right target class
— the storefront sample was just the wrong (gated) class. Tool nuance: for GitHub use
`gh api releases` (downloads are click-triggered, so `api-intercept.py` won't catch them
on page load — its executable bucket is for direct-download / file-host *pages*, still
to be tested on a concrete safe target). **Never download the assets** — names only.
